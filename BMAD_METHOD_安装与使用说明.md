# BMAD Method 安装与使用说明

本文分为两部分：

- **第一部分：给人看**——面向对 AI 技术有了解、但不是专业开发者的读者。
- **第二部分：给 Cursor 看**——让 Cursor 可以按固定步骤在当前项目中检查并安装 BMAD Method。

---

## 第一部分：给非专业开发者看的极简说明

### 1. BMAD Method 是什么？

你可以把 BMAD Method 理解为：

> **给 AI 编程加上一层“项目经理 + 流程导航”的方法。**

如果你直接让 AI 写代码，常见问题是：

- 写着写着跑偏
- 前后说法不一致
- 改一个地方影响另一个地方
- 代码有了，但计划、边界和上下文不清楚

BMAD Method 的作用，就是把这件事变得更有秩序。它强调：

- 先理解问题
- 再规划任务
- 然后分步执行
- 在执行中持续校验

你可以把它看成 AI 开发时的“电子导游”或“项目管家”。

---

### 2. 它适合什么人？

适合这几类人：

- 会和 AI 对话，但不一定是专业程序员
- 想让 AI 帮自己做产品、脚本、工具、小网站
- 已经有一个项目，但越做越乱，想重新找回掌控感
- 想降低 AI“自由发挥过头”的概率

它**不只是适合全新的空白项目**。

对于已经有代码、但文档不完整的项目，BMAD 反而很有价值，因为它能帮助你：

- 让 AI 先整理上下文
- 给后续修改建立计划边界
- 在重构时减少失控

一句话概括：

> **空白项目里，BMAD 像开工前的施工图；已有项目里，BMAD 像接手旧工程时的总控台。**

---

### 3. 安装前需要什么？

先确保你的电脑已经安装：

- **Node.js 20 或更高版本**

在终端中输入下面的命令检查：

```bash
node -v
```

如果看到版本号，并且版本是 20 或以上，就可以继续。

---

### 4. 最简单的安装方法

进入你的项目文件夹后，在终端运行：

```bash
npx bmad-method install
```

然后按提示操作即可。

如果你是在 **Cursor、Claude Code** 等 AI 编程工具里用，它会把所需的规则、工作流、角色模板等内容安装到项目里。

README_CN 中可确认的关键信息：

- 基本安装命令是 `npx bmad-method install`
- 需要 Node.js 20+
- 支持在 Claude Code、Cursor 等工具中使用
- 核心模块 BMM 提供 **34+ 工作流**
- 不知道下一步时，可以直接使用 `bmad-help`

---

### 5. 安装完成后怎么用？

如果你只记一个动作，那就是：

```text
bmad-help
```

当你不知道下一步该做什么时，直接问它。

例如：

```text
bmad-help 我想加一个登录功能，该怎么开始？
```

或者：

```text
bmad-help 我已经完成架构设计，下一步做什么？
```

BMAD 会根据当前阶段，引导你继续走下一步流程。

---

### 6. 最推荐的新手使用方式

对于非专业开发者，最简单的三步是：

#### 第一步：先问路

在 AI 对话框里输入：

```text
bmad-help
```

或者说出你的目标：

```text
bmad-help 我想做一个简单的后台管理页面
```

#### 第二步：让 AI 先写计划

不要一上来就让 AI 大段写代码。更稳妥的方式是先让它生成一个计划文件，例如：

```text
按照 BMAD 方法，先帮我把这个需求拆成一个 plan.md
```

这个 `plan.md` 可以理解为：

- 你和 AI 的任务清单
- 本次开发的边界说明
- 防止 AI 中途跑偏的“合同”

#### 第三步：按计划逐项执行

等计划出来后，再继续说：

```text
根据 plan.md，先执行第 1 步
```

或者：

```text
根据 plan.md，开始开发 Task 1
```

这样会比“直接写完整功能”稳定很多。

---

### 7. 对已有项目，应该怎么用？

如果你的项目已经有很多代码，但缺少文档，不建议一上来就补全所有文档。

更实际的做法是：

#### 做法 A：先生成一份上下文文档

让 AI 先扫描项目，输出一份摘要，例如：

```text
使用 BMAD 规范，扫描当前项目，生成一份 BMAD_CONTEXT.md，说明核心模块、入口、配置结构和主要流程。
```

作用是：

- 先帮 AI 建立“项目地图”
- 也帮你自己重新获得对项目的理解

#### 做法 B：从现在开始，强制先写 plan

以后每做一个需求、修一个 bug、做一次重构，都先让 AI 产出 plan.md。

例如：

```text
根据当前项目上下文，为“修复登录流程中的验证码问题”创建一个 BMAD 风格的 plan.md
```

#### 做法 C：把 BMAD 用在重构和审查上

你可以让它：

- 扫描现有代码，识别风险点
- 给出重构步骤
- 找出测试缺口
- 帮你把“能跑但不敢动”的代码拆解清楚

所以，对于已有项目，更合适的理解不是“全面重建”，而是：

> **局部介入，优先帮你建立上下文和计划边界。**

---

### 8. 在 Cursor 里怎么用？

在 Cursor 中，BMAD 的体验通常是最自然的，因为你安装后，Cursor 会读取项目中的规则与文档。

常见流程如下：

#### 安装

无论你在 **macOS / Linux / Windows**，核心思路都一样：

1. 用 Cursor 打开你的项目根目录
2. 打开 Cursor 内置终端
3. 在项目根目录运行：

```bash
npx bmad-method install
```

如果安装程序让你选择工具，选择 **Cursor**。

##### Windows 环境说明

如果你在 Windows 上使用 Cursor：

- 可以直接在 Cursor 自带终端里运行：

```powershell
npx bmad-method install
```

- 前提仍然是你已经正确安装：
  - Node.js 20+
  - `node -v` 可正常返回版本号
  - `npx` 可正常使用

如果你不确定环境是否正常，可以先在 Cursor 终端执行：

```powershell
node -v
npx --version
```

如果这两个命令都能正常返回结果，再继续安装。

##### Windows 下给 Cursor 的执行要求

如果这部分是给 Cursor 自己读取并自动执行，请按下面原则处理：

1. 当前环境如果是 Windows，也仍然在**当前项目根目录**执行：

```powershell
npx bmad-method install
```

2. 不要因为是 Windows 就改成别的安装逻辑；除非安装程序本身报错，否则仍以 `npx bmad-method install` 为标准入口。

3. 如果安装过程需要交互选择工具，请选择：

- `cursor`

4. 如果项目已经存在：

- `_bmad/`
- `.cursorrules`
- `.cursor/rules/`

则先检查是否为部分安装，不要盲目覆盖。

5. 如果 Windows 环境下执行失败，先报告失败信息，并优先检查：

- Node.js 是否已安装且版本是否 ≥ 20
- `npx` 是否可用
- 当前终端是否位于项目根目录
- 是否因为权限、网络或企业环境限制导致安装中断

#### 使用

安装完成后，打开 Cursor 的 Chat 或 Composer，然后直接输入：

```text
@bmad-help 我想给项目增加一个功能，我应该怎么开始？
```

如果你的实际环境中不是 `@bmad-help`，也可以直接输入：

```text
bmad-help 我想给项目增加一个功能，我应该怎么开始？
```

#### 推荐工作流

1. 先问 `bmad-help`
2. 再生成 `plan.md`
3. 再让 Cursor 根据 plan 逐项执行

---

### 9. 一个最实用的结论

如果你只想记住一句话，那就是：

> **遇事不决，先用 `bmad-help`；开始动手前，先让 AI 写 `plan.md`。**

这两步已经能解决大多数 AI 编程“越写越乱”的问题。

## 第二部分：给 Cursor 读取的安装与执行指令

下面这部分面向 Cursor。目标是：**在当前项目中检查 BMAD 是否已存在；如果未完整安装，则补装；安装后给出可立即执行的下一步。**

---

### A. 目标

请在**当前项目根目录**执行 BMAD Method 的检查与安装。

当前项目的处理原则：

1. **优先检查是否已经部分安装**，不要盲目重复安装。
2. 如果只存在部分文件，则补齐缺失部分。
3. 如果尚未安装，则使用标准方式安装。
4. 安装完成后，输出一段可直接继续使用的建议命令。

---

### B. 当前项目已知状态

根据现有仓库状态，当前项目：

- **已经存在 `_bmad/` 目录**
- **未发现 `.cursorrules` 文件**
- **未发现 `.cursor/` 规则目录**
- `docs/` 目录存在，可用于放置说明文档

这意味着：

> 当前项目**可能已经有部分 BMAD 内容**，但**不一定完成了 Cursor 所需的完整安装**。

因此，正确策略不是直接覆盖，而是：

1. 先检查 BMAD 是否完整可用
2. 再决定是否执行安装命令

---

### C. Cursor 执行步骤

请按以下顺序执行。

#### 第 1 步：检查 BMAD 相关文件是否已存在

检查以下路径是否存在：

- `_bmad/`
- `.cursor/`
- `.cursor/rules/`
- `.cursorrules`
- 其他由 `npx bmad-method install` 生成的工具配置文件

如果满足以下任一条件：

- 已存在 `.cursorrules`
- 已存在 `.cursor/rules/`

则说明 **Cursor 侧规则大概率已经安装过**。此时不要直接重装，先进入“校验模式”。

#### 第 2 步：如果只存在 `_bmad/`，但没有 Cursor 规则

说明：

- BMAD 资源可能已下载到项目中
- 但 Cursor 未完成接入

此时请在项目根目录尝试执行标准安装：

```bash
npx bmad-method install
```

如果安装过程要求选择工具，请选择：

- `cursor`

如果安装工具支持非交互参数，则优先使用能明确指定 Cursor 的方式；否则使用交互式安装。

#### 第 3 步：如果 BMAD 完全不存在

直接在当前项目根目录执行：

```bash
npx bmad-method install
```

并在安装过程中选择：

- 工具：`cursor`
- 模块：优先选择核心模块（如 BMM）

#### 第 4 步：安装完成后校验

安装后再次检查以下内容是否出现：

- `.cursorrules`，或
- `.cursor/rules/`，或
- 其他明确供 Cursor 自动读取的规则文件

如果这些文件仍未出现，请不要假设安装成功，应明确报告：

- 哪一步执行了
- 终端返回了什么
- 目前项目中仍缺失什么

---

### D. 安装成功后的默认使用动作

如果安装成功，请在回复中直接建议用户下一步执行以下任一命令或提示词：

```text
bmad-help
```

```text
bmad-help 我想基于当前项目开始整理开发计划
```

对于已有项目，优先建议的下一步不是直接写代码，而是：

```text
使用 BMAD 规范，扫描当前项目并生成一份 BMAD_CONTEXT.md，说明核心模块、入口、配置结构和主要流程。
```

然后继续：

```text
根据当前项目上下文，为我接下来的需求创建一个 BMAD 风格的 plan.md
```

---

### E. Cursor 执行时的注意事项

1. **不要先入为主地假设当前项目是空白项目。**
   当前项目已经存在 `_bmad/`，说明可能做过部分接入。

2. **不要无条件覆盖已有规则。**
   如果发现已有 `.cursorrules` 或 `.cursor/rules/`，应先提示可能存在规则冲突。

3. **如果安装程序会覆盖旧规则，先提示并征求确认。**
   特别是当项目里已经存在自定义 Cursor 规则时。

4. **如果安装成功，优先进入“上下文整理 + 计划生成”模式。**
   也就是先生成项目上下文，再生成 plan，而不是立刻大规模改代码。

---

### F. Cursor 可直接遵循的简化决策

如果你是 Cursor，请按下面的逻辑执行：

#### 情况 1：发现 `.cursorrules` 或 `.cursor/rules/`

输出：

- BMAD 的 Cursor 规则可能已经存在
- 先校验，不立即重装
- 然后建议用户直接使用 `bmad-help`

#### 情况 2：发现 `_bmad/`，但没发现 Cursor 规则

输出：

- 当前项目可能只完成了 BMAD 资源层安装，尚未完成 Cursor 接入
- 需要执行 `npx bmad-method install`
- 安装时选择 `cursor`

#### 情况 3：三者都不存在

输出：

- 当前项目尚未安装 BMAD Method
- 在项目根目录执行 `npx bmad-method install`
- 安装时选择 `cursor`

---

### G. 给 Cursor 的最终执行提示词

如果需要一个单段提示词，可直接使用下面这段：

```text
请检查当前项目是否已经安装 BMAD Method。重点检查 _bmad/、.cursorrules、.cursor/rules/ 是否存在。如果已经有 Cursor 规则，则不要直接重装，先告诉我当前状态；如果只有 _bmad/ 但没有 Cursor 规则，请在项目根目录执行 npx bmad-method install，并在安装过程中选择 cursor；安装完成后再次校验规则文件是否出现。成功后，不要直接开始写代码，而是先建议我运行 bmad-help，并优先生成 BMAD_CONTEXT.md 和 plan.md。
```
